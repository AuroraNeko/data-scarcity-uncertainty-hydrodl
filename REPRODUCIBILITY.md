# Reproducibility Guide

This document records the information needed to reproduce and audit the results
reported in:

**How Data Scarcity Compromises Uncertainty Estimates in Hydrological Deep
Learning and How Conformal Calibration Mitigates It**.

The public repository supports two reproducibility levels:

1. Rerun the experiments after setting up CAMELS-US locally.
2. Audit the reported figures and tables from the stored JSON/CSV result files.

The second path is the fastest way to verify that the reported numbers match
the machine-readable outputs committed with this repository.

Archived code and result snapshot:
https://doi.org/10.5281/zenodo.21256956

## 1. Environment

The reference experiments were run with Python 3.11, PyTorch 2.x, and a single
NVIDIA RTX 5060 Ti GPU. Training is GPU-recommended. Figure generation and
result verification can run on CPU once the result files are present.

Install dependencies with:

```bash
pip install -r requirements.txt
```

The required dependency families are PyTorch, NumPy, pandas, matplotlib,
XGBoost, and SciPy.

## 2. Data

The study uses CAMELS-US version 2.0 with 671 basins. The preprocessing pipeline
uses three meteorological forcing products: Daymet, Maurer, and NLDAS. From each
forcing product, five daily variables are used: precipitation, minimum and
maximum temperature, solar radiation, and vapor pressure. The model also uses 13
static catchment attributes covering topography, climate, soil, vegetation, and
geology.

The download helper points to the CAMELS-US archive used by this project:

- Zenodo record: https://zenodo.org/records/15529996
- CAMELS-US DOI: https://doi.org/10.5065/D6MW2F4D

Download and preprocess the data with:

```bash
python download_camels.py
python src/data/data_preprocessing.py
```

The processed basin files are expected under:

```text
data/processed/camels_us/
```

Raw and processed CAMELS-US data are not tracked in git because of their size
and upstream data-distribution policy. The canonical data settings are
summarized in `configs/data_config.yaml`.

## 3. Temporal Splits

The splits are water-year aligned:

| Split | Period | Role |
|---|---|---|
| Training | 1 Oct 1980 to 30 Sep 1995 | Model fitting; truncated to 1, 3, or 5 water years for scarcity experiments |
| Validation / calibration | 1 Oct 1995 to 30 Sep 2000 | CQR calibration and validation monitoring for point baselines |
| Test | 1 Oct 2000 to 30 Sep 2010 | Final held-out evaluation |

## 4. Main Experimental Settings

| Component | Setting |
|---|---|
| Scarcity basin subset | 50 basins sampled once with NumPy `RandomState(42)` and reused for all scarcity durations |
| Sequence length | 365 days for full 15-year models; 30, 90, 180, and 365 days for 1-, 3-, 5-, and 15-year scarcity runs |
| Quantile levels | 0.05, 0.50, 0.95 |
| CQR target | 90% intervals, `alpha = 0.10` |
| Optimizer | Adam, learning rate 0.001, betas 0.9 and 0.999 |
| Batch size | LPU-Stream full-dataset quantile/ensemble runs use 1024; scarcity runs use 512 for 1-/3-year models and 1024 for 5-/15-year models; point baselines use model-specific values in `experiments/baseline/train_model.py` |
| Gradient clipping | 1.0 |
| Early stopping | patience 5, maximum 30 epochs |
| Deep Ensemble seeds | 42, 123, 456, 789, 999 |
| 1-year stability seeds | 42, 123, 456 |
| Bootstrap | Basin-cluster bootstrap with 500 resamples for full-dataset uncertainty metrics |

## 5. Reproduction Path

Run the download and preprocessing steps first:

```bash
python download_camels.py
python src/data/data_preprocessing.py
```

After the processed CAMELS-US files are available, the experiment stages can be
launched with:

```bash
python experiments/orchestrator.py
```

The orchestrator is resumable: it checks expected outputs and skips stages that
are already complete. It assumes data preprocessing has already been done and
expects large model checkpoints under `results/checkpoints/` while training is
in progress. Checkpoints are intentionally not tracked in git; they can be
regenerated by rerunning the corresponding training stages.

The main stages can also be run manually:

```bash
python experiments/baseline/train_model.py --model lpu_stream
python experiments/baseline/train_xgboost.py
python experiments/analysis/eval_point_perbasin.py

python experiments/uncertainty/train_quantile.py
python experiments/uncertainty/retrain_ensembles_correct.py
python experiments/uncertainty/eval_fair_671.py

python experiments/scarce/train_data_scarce.py --years 1
python experiments/scarce/train_data_scarce.py --years 3
python experiments/scarce/train_data_scarce.py --years 5
python experiments/scarce/train_data_scarce.py --years 15

python experiments/analysis/diagnose_1yr.py
python experiments/analysis/cross_region_validation.py
python experiments/analysis/stability_1yr.py
python experiments/analysis/confidence_levels.py
python experiments/analysis/enhanced_robustness.py
python experiments/analysis/basin_representativeness.py
python experiments/analysis/make_figures.py
```

## 6. Fast Audit Path

The result files used for the reported tables and figures are stored under:

```text
results/tables/
results/figures/
```

Regenerate figures from stored tables:

```bash
python experiments/analysis/make_figures.py
```

Verify that reported numbers match the stored JSON/CSV outputs:

```bash
python experiments/analysis/verify_manuscript.py
```

If `paper/manuscript.tex` is present in a local author checkout, the
verification script also checks selected text strings. In the public code
checkout, it skips those text checks and still audits the machine-readable
result files. A fully passing run reports `ALL CHECKS PASSED`.

## 7. Public Repository Contents

The public code repository is intended to include:

| Artifact | Purpose |
|---|---|
| `src/` and `experiments/` | Model, training, uncertainty, and analysis code |
| `configs/data_config.yaml` | Canonical data split and preprocessing reference |
| `requirements.txt` | Python dependency families |
| `results/tables/*.json` and `*.csv` | Machine-readable numerical results |
| `results/figures/*.pdf` and `*.png` | Generated figures |
| `CITATION.cff` | Citation metadata for GitHub and Zenodo |
| `README.md` and `REPRODUCIBILITY.md` | Public usage, audit, and rerun instructions |

The public code repository intentionally excludes:

| Excluded path or artifact | Reason |
|---|---|
| `/data/` | Raw and processed CAMELS-US files are large and should be obtained from the upstream archive |
| `/results/checkpoints/` | Model checkpoints are large training artifacts and can be regenerated |
| `/paper/` and `/submission/` | Paper source, compiled PDFs, cover letters, and submission materials are managed separately from the code release |
| `/pdf/` | Local reference PDFs may be copyrighted third-party material |
| `*.log` and local workspace state | Machine-specific run traces are not part of the reproducible public artifact |

## 8. Traceability Notes

The reported method comparison uses the full 671-basin model. The matched
data-scarcity gradient uses one fixed 50-basin seed-42 subset. The committed
files `results/tables/scarcity_50_basin_ids.csv` and
`results/tables/basin_representativeness_summary.csv` document that subset and
its representativeness across the tested static attributes.

The verification script links the stored numerical outputs to the reported
tables, figures, splits, seeds, sequence lengths, calibration settings, and
robustness checks.
